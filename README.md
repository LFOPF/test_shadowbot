# ShadowBot

Асинхронный Telegram-бот для перевода веб-новелл с фокусом на **максимальное качество художественного перевода**, **оптимизацию пайплайна обработки** и **отказоустойчивость в проде (Railway, один контейнер)**.

## Ключевые цели проекта

1. **Качество перевода**
   - двухпроходный пайплайн (черновой перевод + финальная литературная редактура);
   - глоссарий терминов для консистентности имён, титулов и лора;
   - очистка и валидация контента перед отправкой в LLM и перед публикацией.

2. **Оптимизация процессов перевода**
   - агрессивное кэширование в Redis (метаданные, оригинал, перевод, URL Telegraph);
   - блокировки на уровне главы и пользователя, чтобы не переводить одно и то же несколько раз;
   - минимизация лишних сетевых запросов и повторных операций.

3. **Отказоустойчивость**
   - fail-fast проверки на старте;
   - retry для внешних API с разделением временных и невосстановимых ошибок;
   - защитные механизмы парсинга и публикации;
   - управление браузером Playwright и ограничение параллелизма.

## Что делает бот

- парсит главы (включая JS-страницы) через Playwright;
- находит и валидирует нужную главу;
- переводит контент через OpenRouter;
- публикует результат в Telegraph;
- уведомляет подписчиков о новых главах;
- поддерживает закладки и пользовательские сценарии в Telegram.

## Архитектура (без лишней сложности)

Проект работает как **один процесс aiogram-бота** с Redis и внешними API.

- **Telegram / aiogram** — диалоги, кнопки, FSM, админ-сценарии.
- **Redis** — состояние, кэш, блокировки, статусы обработки.
- **Playwright** — загрузка и разбор страниц глав.
- **OpenRouter** — генерация перевода и редактура.
- **Telegraph** — публикация читабельной версии перевода.

> Архитектура намеренно простая: один контейнер, запуск `python bot.py`, без дополнительных сервисов.

## Поток обработки главы

1. Пользователь запрашивает главу.
2. Бот проверяет кэш и метаданные в Redis.
3. Если результат уже готов — сразу отдаёт ссылку.
4. Если нет — берёт lock по главе и запускает обработку.
5. Парсит страницу, проверяет валидность извлечённого контента.
6. Прогоняет двухэтапный перевод OpenRouter.
7. Публикует результат в Telegraph.
8. Сохраняет кэш и отдаёт ссылку всем ожидающим запросам.

## Надёжность: что важно сейчас

### 1) Fail-fast startup checks

Перед полноценным стартом бот проверяет:
- `system_prompt.txt` существует и не пустой;
- `user_prompt.txt` существует и не пустой;
- `glossary.txt` читается;
- Redis доступен.

Если проверка не проходит — процесс завершается сразу, не дожидаясь «падения на первом запросе».

### 2) Устойчивый поиск глав при росте сайта

Основной быстрый путь — бинарный поиск в пределах `MAX_PAGES`.
Если предел достигнут и глава потенциально находится глубже, используется **ограниченный fallback** расширенного диапазона, чтобы:
- не терять главы при росте каталога;
- не создавать избыточную нагрузку на сайт.

### 3) Корректный retry для Telegraph

Ошибки разделены на типы:
- **retriable** (сеть, timeout, временные HTTP/API ошибки) — пробрасываются для retry;
- **non-retriable** (битый payload и постоянные ошибки) — останавливаются без бесполезных повторов.

`TITLE_TOO_LONG` обрабатывается отдельно: повторная публикация с укороченным заголовком.

## Переменные окружения

### Обязательные
- `BOT_TOKEN`
- `OPENROUTER_API_KEY`
- `TELEGRAPH_ACCESS_TOKEN`
- `REDIS_URL`

### Дополнительные
- `ADMIN_ID`
- `BROWSER_IDLE_TIMEOUT`
- `PLAYWRIGHT_CONCURRENCY`
- `GLOSSARY_CACHE_TTL`
- `USER_CHAPTER_REQUEST_LOCK_TTL`
- `USER_CHAPTER_CANCEL_TTL`
- `OPENROUTER_TRANSLATION_MODEL`

## Локальный запуск

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
playwright install chromium-headless-shell
python bot.py
```

## Тесты

Быстрый запуск основных тестов:

```bash
python -m pytest -q tests/test_resilience.py
```

(Тесты используют моки и не требуют реальных вызовов OpenRouter/Telegraph/Playwright.)

## Docker / Railway

Проект рассчитан на Railway в текущем формате:
- один контейнер;
- существующий `Dockerfile`;
- запуск `python bot.py`.

Сборка и запуск локально:

```bash
docker build -t shadowbot .
docker run --env-file .env shadowbot
```
